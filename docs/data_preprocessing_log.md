# 데이터 전처리 로그 (data_preprocessing_log.md)

## 1. 프로젝트명
전환교육 공백 분석 및 맞춤 경로 추천 시스템 (작동형 웹 프로토타입)

## 2. 작성 목적
원자료(공공데이터 또는 가상 CSV)를 본 프로토타입의 시연용 JSON으로 변환하는 과정을 기록한다. 산식·가중치·분류 기준을 외부 검증 가능한 형태로 남겨, 발표자료의 신뢰성과 재현 가능성을 보강한다. 본 문서는 발표자료 5·6·14·15쪽과 직접 연결된다.

## 3. 데이터 구분 (가장 중요한 원칙)

본 시스템은 두 종류의 데이터를 **명확히 구분해서** 사용한다. 발표·시연 시 두 데이터를 혼동하지 않도록 주의한다.

| 구분 | 무엇인가 | 활용 목적 | 사용 화면 |
|------|---------|-----------|-----------|
| **연도별 데이터** (yearlySupport) | 최근 5년간 자원·접근성·상담 점수의 변화 시계열 | **교육청 정책 판단** (자원 변화 추세 식별) | 지역별 공백 분석 (LineChart), 대시보드(추세위험도 카드) |
| **현재 기준 데이터** (currentYear=2026) | 가장 최근 연도의 자원·접근성·상담·연계 점수 | **교사 상담 및 학생 맞춤 추천** (즉시 활용 가능한 자원 매칭) | 대시보드(공백지수·공백 유형), 지역별 공백 분석(공백 유형), 맞춤 경로 추천 |

### 3.1. 연도별 데이터의 활용 목적
- 최근 5년간 프로그램 수, 연계기관 수, 접근성 점수, 상담지원 점수, 진로연계 점수의 추세 추적
- 자원 감소가 큰 지역을 **추세위험도(보조지표)** 로 식별
- 교육청 정책 우선순위 판단(자원 재배분, 협약 갱신, 셔틀 운영 등)에 사용
- 단기적인 학생 추천에는 사용하지 않음 (학생 추천은 항상 현재 기준 데이터로)

### 3.2. 현재 기준 데이터의 활용 목적
- 가장 최근 연도(2026년)의 자원·접근성·상담·연계 점수를 기반으로 **현재 공백지수(주지표)** 산출
- 가장 부족한 항목을 기준으로 **현재 공백 유형** 분류 (프로그램 부족형 / 기관 부족형 / 접근성 취약형 / 상담지원 부족형 / 진로연계 약화형)
- 교사가 학생에게 즉시 추천 가능한 자원 목록 작성에 사용
- 정책 판단에 직접 사용하지 않음 (정책 판단은 5년 추세를 함께 보아야 함)

## 4. 지표의 시각적 위계

| 위계 | 지표 | 데이터 출처 | 화면 표현 |
|------|------|-------------|-----------|
| **주지표** | 현재 공백지수 (currentGapIndex) | 현재 기준 데이터 | navy 배경 큰 카드 (60px 폰트), 0~100 |
| **보조지표** | 추세위험도 (trendRiskScore) | 연도별 데이터 (5년 변화율 기반) | 흰색 작은 카드 (30px 폰트), 0~100 |
| 분류 | 현재 공백 유형 (gapType) | 현재 기준 데이터 | 배지 형태 |
| 추세 시각화 | yearlySupport | 연도별 데이터 | Recharts LineChart |

> **현재 공백지수는 주지표, 추세위험도는 보조지표**임을 발표 시 반드시 강조한다. 이는 본 시스템의 차별성과 직결되는 핵심 메시지다.

## 5. 원자료 컬럼 설명

| 컬럼 | 의미 | 단위 | 비고 |
|------|------|------|------|
| year | 자료 기준연도 | 연 | 2022~2026 (5년) |
| region | 지역명(권역) | - | 표준화 대상 (섹션 7 참고) |
| student_demand | 권역 내 전환교육 대상 학생 추정 수 | 명 | 결측 시 보간 |
| program_count | 권역 내 운영 프로그램 수 | 개 | 정수 |
| agency_count | 권역 내 연계기관 수 | 개 | 정수 |
| accessibility_score | 접근성 점수 | 0~100 | 거리·교통·온라인 결합 |
| counseling_support_score | 상담지원 점수 | 0~100 | 전담 상담인력·회차 결합 |
| career_linkage_score | 진로연계 점수 | 0~100 | 외부기관·졸업 후 경로 결합 |

## 6. 전처리 단계 (Step-by-Step)

전처리는 [scripts/preprocess.py](../scripts/preprocess.py)에 구현되어 있으며, 다음 순서로 동작한다.

### Step 1. 원자료 CSV 읽기
- 입력: `data/raw/transition_support.csv` (UTF-8)
- `pd.read_csv(encoding="utf-8")`

### Step 2. 지역명 표준화 — `standardize_region_name(region)`
- 공백 표기 통일: `'서울A권역'` → `'서울 A권역'`
- `'권 역'` → `'권역'` 등 띄어쓰기 변형 제거
- 권역 코드와 권역 명칭이 1:1 매핑되도록 정규화
- 표기 차이로 인한 권역 분리 방지

### Step 3. 숫자형 변환 — `pd.to_numeric(errors="coerce")`
- 모든 수치 컬럼을 강제 숫자형 변환
- 변환 실패값(예: `"N/A"`, `"-"`)은 결측치(NaN)로 처리

### Step 4. 결측치 처리
1. 같은 지역의 다른 연도 평균으로 1차 보간
2. 그래도 남는 결측은 0으로 처리
3. (실제 적용 시 권장) `student_demand` 결측이 50% 이상인 지역은 분석 대상에서 제외

### Step 5. 연도별 데이터와 현재 기준 데이터 분리
- 연도별 데이터: 전체 5년치 행을 그대로 보존 → JSON의 `yearlySupport` 배열
- 현재 기준 데이터: `df[df["year"] == max(year)]` 필터링 → 공백지수·공백 유형 산출의 입력

### Step 6. 정규화 — `normalize_series(series, reverse=False)`
- 각 시리즈를 0~100 범위로 min-max 정규화
- 자원 비율(높을수록 좋음)·점수(높을수록 좋음)는 `reverse=True` 적용 → "값이 클수록 부족도가 큼"으로 변환

### Step 7. 현재 공백지수 계산 — `calculate_current_gap_index(current_df)`

```
gap_index = 0.30 * program_gap
          + 0.20 * agency_gap
          + 0.20 * accessibility_gap
          + 0.15 * counseling_gap
          + 0.15 * career_linkage_gap
```

- `program_gap`, `agency_gap`: 학생 수요 대비 자원 비율을 0~100 정규화 후 reverse
- 점수형 컬럼은 0~100 정규화 후 reverse
- 결과: 0~100, **값이 클수록 공백이 큼**

> ⚠️ **본 산식의 가중치(0.30 / 0.20 / 0.20 / 0.15 / 0.15)는 프로토타입용 예시 산식입니다.**
>
> 실제 서비스 적용 시 **교육청·특수교육 전문가·데이터 전문가의 검토**를 거쳐 가중치, 정규화 방식, 결측치 처리 규칙을 보정해야 합니다.

### Step 8. 추세위험도 계산 — `calculate_support_change()`, `calculate_trend_risk_score()`

```
support_change(%) = (마지막 연도 자원 합 - 첫 연도 자원 합) / 첫 연도 자원 합 * 100
trend_risk_score   = clamp(50 - support_change, 0, 100)
```

- 자원 합: `program_count + agency_count + accessibility_score / 10`
- 자원이 감소(`support_change < 0`)할수록 큰 값
- 결과: 0~100

### Step 9. 현재 공백 유형 분류 — `classify_gap_type(row)`

현재 기준 데이터에서 다음 항목 중 가장 낮은 항목을 공백 유형으로 분류:

| 가장 낮은 항목 | 공백 유형 |
|----------------|-----------|
| program_count | 프로그램 부족형 |
| agency_count | 기관 부족형 |
| accessibility_score | 접근성 취약형 |
| counseling_support_score | 상담지원 부족형 |
| career_linkage_score | 진로연계 약화형 |

> ⚠️ 본 분류 규칙도 예시이며, 실제 적용 시 동률·복합 유형 처리 로직 추가가 필요합니다.

### Step 10. JSON 변환 및 저장 — `build_region_json(df)` → `regions.json`
- 출력: `src/data/regions.json`
- 각 지역 항목 스키마:

| 키 | 의미 | 출처 |
|----|------|------|
| `region` | 지역명 | 표준화된 region |
| `currentYear` | 기준연도 | 2026 |
| `yearlySupport` | 5년치 연도별 데이터 배열 | Step 5 |
| `currentGapIndex` | 현재 공백지수 (주지표) | Step 7 |
| `trendRiskScore` | 추세위험도 (보조지표) | Step 8 |
| `supportChange` | 5년 변화율(%) | Step 8 |
| `gapType` | 현재 공백 유형 | Step 9 |
| `mainIssue` | 주요 원인 설명 | 텍스트 자동 생성 + 팀 검토 |
| `policyUse` | 교육청 정책 활용 방향 | 텍스트 자동 생성 + 팀 검토 |
| `teacherUse` | 교사 상담 활용 방향 | 텍스트 자동 생성 + 팀 검토 |

## 8. 한계 및 보완 계획
1. **산식 검증 필요**: 공백지수·추세위험도 가중치는 예시이며, 교육청·전문가 검토를 거쳐 보정해야 한다.
2. **표본 크기**: 현재는 4개 권역의 시연용 데이터만 포함. 실제 적용 시 전국 권역 단위 데이터로 확장한다.
3. **장기 추세**: 5년 자료만 사용. 정책 효과 측정에는 더 긴 시계열이 권장된다.
4. **개인 단위 데이터 부재**: 본 프로토타입은 개인정보를 저장하지 않으며, 추천은 권역·프로필 매칭 수준의 시연이다. 실제 서비스에서는 개별 학생 데이터 활용 시 별도 동의 절차가 필요하다.
5. **공식 데이터 연계**: 실제 공공데이터(교육부·시도교육청·NICE 등)와 연계 시 컬럼명 매핑 규약과 갱신 주기 정의가 필요하다.
6. **지표의 의미 검증**: 공백지수가 0~100 범위에서 어느 임계값부터 "심각"으로 분류되는지의 정의가 필요하다 (현재는 단순 정규화 결과).
7. **분류 동률 처리**: 현재 공백 유형 분류는 동률 발생 시 한 항목만 선택. 복합 유형 표현 방식 검토 필요.
